# `_兴趣挖掘日志/` 归档规范

按主题归档历史兴趣挖掘记录。文件名采用**内容总结式命名**（如 `股市交易兴趣生长记录.md`、`分形生成艺术探索.md`），日期信息记录在文件内部。

模板：

```markdown
# 兴趣挖掘日志 — [内容主题]

> 归档时间：YYYY年M月

## 种子溯源
| 兴趣 | 触发日期 | 去向 |
|------|----------|------|
| 分形图形 | 2026-07-17 | → 探索中：[[分形与生成艺术]] |
| 编译器原理 | 2026-07-05 | → 暂停（等学完 Python 类） |
| 游戏开发 | 2026-07-10 | → 放弃（兴趣消退） |

## 详细记录
### 2026-07-17
- 在 [[Python 基础/课程/05_递归]] 的检查站，你提到"能不能用递归画雪花曲线？很想试一下"。
  触发概念：递归、分形、科赫雪花
  → 已创建探索主题：[[分形与生成艺术]]
```

> **命名原则**：文件名应让用户一眼看出内容范围，避免按日期打开时需要猜测里面是什么。一个日志文件可以覆盖一个主题方向在所有时间产生的兴趣记录。
