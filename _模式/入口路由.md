# 入口路由

## 八种入口速查表

| 用户表达 | 模式 | 行动 |
| --- | --- | --- |
| 我想学 X / 学习 X / 教我 X | 学习模式 | 新建或继续主题，生成课程与检查站 |
| 继续 X / 恢复 X | 恢复模式 | 检查暂停主题的复归条件，满足则恢复学习 |
| 我想复习 X / 复习 X | 复习模式 | 打开对应主题，读取遗漏和复习计划 |
| 今天该复习什么 | 全局复习模式 | 读取 `_全局索引.md`，输出到期/过期项 |
| 看看我的错题 | 错题模式 | 读取所有主题的 `错题/_活跃遗漏.md` |
| 我多久没学过 X 了 | 学习间隔查询 | 读取对应 `进度.md`，返回距上次学习/复习的天数 |
| 快速过一遍 X | 微复习模式 | 仅抽取遗漏点和上次卡点做5分钟快问快答 |
| 今天不学了 / 休息一下 / 暂停一下 | 休息模式 | 更新进度.md「最后学习日期」为今天，不做其他操作 |

## 学习中提问的特别处理

当用户**已在某门课程的学习中**（会话上下文正在讲某节课），发出以下类型的提问时，**不切换模式**——在当前学习模式内处理：

- "XX是什么"、"XX是什么意思"、"什么叫XX" → 回答概念 → 如果概念深度足够，触发笔记自动创建（见 `学习模式.md` 2.5 节）
- "这个我不太懂"、"再解释一下XX" → 苏格拉底追问，不切换模式
- "举个XX的例子" → 在当前课程上下文中给出例子

这些不会触发路由重判，助手应在学习模式内部自然响应。

## 路由优先级分组

路由按以下顺序匹配，命中后立即停止：

### 第一组：新会话入口（最高优先级）
用户刚打开会话，说"我想学XX" → 学习模式（新建主题需先起始评估，已有主题按优先级判断）

### 第二组：全局操作
"今天该复习什么" / "看看我的错题" → 不需要指定主题

### 第三组：主题级操作（需指定 X）
"复习 X" / "继续 X" / "快速过一遍 X" / "休息一下" → 对应模式

### 第四组：学习中提问（最低优先级——不切换模式）
如果用户已经在某个课程中，任何"XX是什么""我不太懂""举个例子"等提问，都**不切换模式**，在当前学习上下文中响应。

## "我想学 X" 的优先级判断

当用户说"我想学 X"且主题文件夹已存在时，按以下优先级：

1. **若 X 是探索中主题**：先读取其 `进度.md` 中的「用户意图倾向」。若倾向为"即将正式开课"，直接询问"要现在正式开课吗？"否则告知当前状态并询问："要把它正式开课，还是继续以探索模式添加灵感笔记？"
2. **若 X 是暂停主题**：按恢复模式处理（见 `恢复模式.md`）。
3. **若 X 是已完成主题**：告知用户"这个主题已标记为完成。要重新激活它吗？"若确认，将其从「暂停/已完成」移回「当前主题」，由用户决定是复习旧课还是扩展新内容。
4. **若 X 是活跃主题**：检查 `复习计划.md` 是否有到期项。有则告知并询问：先复习还是继续新课。无到期项则告知无待复习，直接继续新课，或提议快速回顾要点。
