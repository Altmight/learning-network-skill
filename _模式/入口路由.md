# 入口路由

## 七种入口速查表

| 用户表达 | 模式 | 行动 |
| --- | --- | --- |
| 我想学 X / 学习 X / 教我 X | 学习模式 | 新建或继续主题，生成课程与检查站 |
| 继续 X / 恢复 X | 恢复模式 | 检查暂停主题的复归条件，满足则恢复学习 |
| 我想复习 X / 复习 X | 复习模式 | 打开对应主题，读取遗漏和复习计划 |
| 今天该复习什么 | 全局复习模式 | 读取 `_全局索引.md`，输出到期/过期项 |
| 看看我的错题 | 错题模式 | 读取所有主题的 `错题/_活跃遗漏.md` |
| 我多久没学过 X 了 | 学习间隔查询 | 读取对应 `进度.md`，返回距上次学习/复习的天数 |
| 快速过一遍 X | 微复习模式 | 仅抽取遗漏点和上次卡点做5分钟快问快答 |

## "我想学 X" 的优先级判断

当用户说"我想学 X"且主题文件夹已存在时，按以下优先级：

1. **若 X 是探索中主题**：先读取其 `进度.md` 中的「用户意图倾向」。若倾向为"即将正式开课"，直接询问"要现在正式开课吗？"否则告知当前状态并询问："要把它正式开课，还是继续以探索模式添加灵感笔记？"
2. **若 X 是暂停主题**：按恢复模式处理（见 `恢复模式.md`）。
3. **若 X 是活跃主题**：检查 `复习计划.md` 是否有到期项。有则告知并询问：先复习还是继续新课。无到期项则告知无待复习，直接继续新课，或提议快速回顾要点。

## 模式跳转

- 命中「学习模式」→ 加载 `学习模式.md`
- 命中「恢复模式」→ 加载 `恢复模式.md`
- 命中「复习模式」→ 加载 `复习模式.md`
- 命中「全局复习模式」→ 加载 `全局复习模式.md`
- 命中「微复习模式」→ 加载 `微复习模式.md`
- 命中「错题模式」→ 直接读取所有主题的 `错题/_活跃遗漏.md`
- 命中「学习间隔查询」→ 直接读取对应 `进度.md`，返回距上次学习/复习的天数
