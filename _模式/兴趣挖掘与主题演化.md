# 兴趣挖掘与新主题创建（三层梯度）

兴趣挖掘不只是一个"记下来以后再说"的动作——当发现深度兴趣时，按以下三层梯度处理，在"不延迟"和"不混杂"之间取得平衡：

## 第一层：种子即时捕获（不离开当前主题）

在学习或复习过程中，当用户表达出对某个方向的兴趣时，立即在 `_知识关联图谱.md` 的兴趣挖掘记录中以**完整上下文**记录，而不是只写一行"对XX感兴趣"。格式关键：写明**触发概念**和**当前关联课程**，便于未来反向链接。

```markdown
- YYYY-MM-DD：在 [[主题/课程/NN_标题]] 的检查站，你提到"……（原话）"。
  触发概念：……
  当前关联课程：[[主题/课程/NN_标题]]
  初步方向：……
  → 已记录意向主题：[[新主题名]]
```

## 第二层：意向池升格为"探索主题"（轻量孵化）

当用户不止一次提到同一个兴趣，或主动说"我想详细了解一下XX"时，在 `my-learning-log/` 下为它创建一个**轻量级探索空间**，但不立刻进入完整学习模式（不污染主课程进度）。

探索中主题的最小结构：

```
[主题名]/
├── 进度.md          # 状态标记为"探索中"（使用探索中主题简化模板，含用户意图倾向）
├── 灵感笔记.md       # 链接、想法、随手记录，自由格式
├── 关联.md          # 从哪些主题迁入、可能迁出到哪里
```

`灵感笔记.md` 为自由格式，用户可以随意记录链接、灵感、试写的代码片段等。`关联.md` 专门记录进出关系：

```markdown
# [分形与生成艺术] 的关联

## 迁入
- [[Python 基础]]：递归与海龟绘图
- 数学直觉：对称与自相似

## 可能迁出
- 如果深入数学部分 → [[分形几何]] 或 [[动力系统]]
- 如果偏向艺术创作 → [[生成艺术与创意编程]]
```

在 `_全局索引.md` 的「探索中主题」表格中新增条目（模板见 `_全局结构/全局索引模板.md`），确保不被遗忘。

## 第三层：触发正式开课，并主动编织关联

当用户明确说"开始意向池里的XX"或"把XX正式开课"时：

1. 按学习模式完整初始化主题（补齐 `课程/`、`错题/`、`复习计划.md`，移除 `灵感笔记.md` 或将其内容融入第一课）。
2. 把轻量 `关联.md` 中的内容迁移为正式格式（使用 `_主题内部/关联模板.md`），在 `_知识关联图谱.md` 的跨主题索引中添加一行极简摘要（1 行以内）。
3. **写具体的桥接语句**——不只是罗列链接，而是解释概念之间**为什么相关**：

```markdown
- [[分形与生成艺术/课程/01_自相似性]] 与 [[Python 基础/课程/05_递归]] 中的递归定义直接对应：
  "分形的局部与整体相似"就是递归逻辑的几何体现。
```

4. 更新 `_全局索引.md`：从「探索中主题」移除，新增到「当前主题」。
5. 从 `_知识关联图谱.md` 的「意向主题池」移除该条目。

## 深度兴趣的回退机制

有时探索一阵后发现暂时没时间深入，或需要前置知识。此时把主题状态从"探索中"改为"暂停"，在 `_全局索引.md` 的「暂停/已完成主题」中写一句话说明暂停原因和复归条件（如"等学完 Python 类的知识再重开"）。这能避免"僵尸主题"堆积带来的心理负担。

## 主题结构演化：合并与拆分

随着学习深入，知识网络可能需要结构性的调整。助手应在以下情况下主动提示：

**主题合并**：当两个主题的课程内容或核心概念重叠超过 50%（例如"Python 数据分析"和"统计学入门"越学越像），助手主动建议："我注意到 [主题A] 和 [主题B] 的内容有较大重叠，是否需要合并为一个主题？我可以帮你整理迁移方案。"由用户确认后，将两个主题的课程和错题记录迁移到新主题名下，更新 `_知识关联图谱.md` 和 `_全局索引.md`。

**主题拆分**：当一个主题的课程数超过 **15 课**且内部可以清晰分为两个或多个子领域时（例如"机器学习"可拆分为"监督学习"和"无监督学习"），助手主动建议拆分，并说明如何分配已有课程。由用户确认后创建两个新主题并分配课程，原主题归档为已完成。

## 主题完成判定

当以下两个条件同时满足时，助手主动询问"这个主题是否可以标记为已完成？"：

1. 「正在学习」字段为空（所有已生成的课程均已完成）
2. 用户实际达到的水平与 `进度.md` 中设定的目标程度一致或更高

用户确认后：
- 在 `_全局索引.md` 中将其从「当前主题」移至「暂停/已完成」，状态标记为"已完成"。
- 已完成主题的所有课程视为长期掌握，每 90 天在全局复习模式中提示一次可选回顾。
- 已完成主题仍可随时通过"我想学 X"重新激活——此时将其移回「当前主题」，由用户决定是复习旧课还是扩展新内容。

## 主动提醒规则

- 当用户在检查站或日常对话中表达出对某个方向的兴趣时，立即记入 `_知识关联图谱.md` 的兴趣挖掘记录或意向主题池（第一层）。
- 每学完一个主题的一个阶段，可以主动提醒意向池或探索中主题中是否有值得推进的方向。
