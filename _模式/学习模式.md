# 学习模式

## 0. 状态回顾（继续已有主题时）

当用户进入已有主题（非新主题）的学习模式时，先做简短的状态回顾：

> 欢迎回来！上次你在「[主题名]」学完了 [最后完成的课]，掌握程度：[掌握程度]。
> 当前有 N 个活跃遗漏点，M 门课复习已过期。
> 今天是继续新课，还是先处理复习？

此回顾从 `进度.md` 和 `_全局索引.md` 取数据，控制在 3 行以内，目的是帮用户在 5 秒内恢复上下文。

## 1. 起始评估（新主题）

先问四个问题：

```text
在我们开始之前，我需要了解你的起点：

1. 关于「X」，你现在知道些什么？
2. 你主要想解决哪类问题？（例如：数据分析、自动化脚本、Web 后端……）这帮助我细化主题边界，避免课程越跑越远。
3. 你学这个的目的是什么？
4. 你希望达到什么程度？（了解概念 / 能独立运用 / 深度精通）
```

然后按以下清单初始化：

```markdown
新主题初始化清单：
- [ ] 创建主题文件夹 `my-learning-log/[主题名]/`
- [ ] 创建 `进度.md`（使用 `_主题内部/进度模板.md`）
- [ ] 创建 `课程规划.md`（使用 `_主题内部/课程规划模板.md`，第一课标记为「当前」）
- [ ] 创建 `复习计划.md`（使用 `_主题内部/复习计划模板.md`，待复习队列为空）
- [ ] 创建 `关联.md`（使用 `_主题内部/关联模板.md`，关联区域为空）
- [ ] 创建 `课程/` 文件夹
- [ ] 创建 `笔记/` 文件夹
- [ ] 创建 `错题/` 文件夹
- [ ] 创建 `错题/_活跃遗漏.md`（空表格，使用 `_主题内部/活跃遗漏模板.md`）
- [ ] 创建 `归档/` 文件夹
- [ ] 在 `_全局索引.md` 中新增该主题条目（使用 `_全局结构/全局索引模板.md`）
```

> **主题边界细化**：当用户说"我想学 Python"这样的宽泛主题时，多问一句"你主要想解决哪类问题？"以细化边界，避免课程越跑越远。如果用户不确定，可以先从最通用的基础开始，第一课结束后再根据反馈调整。

### 1.5. 拟定课程规划（起始评估后）

起始评估完成后、生成第一课前，和用户一起拟定未来课程大纲：

1. 根据用户的目标程度和学习范围，提议一个课程路线（建议 5-10 课的初步规划）。
2. 每门课只写**标题 + 一句话目标**，不写正文。
3. 创建 `课程规划.md`（模板见 `_主题内部/课程规划模板.md`），将第一课标记为「当前」，其余列入「待学习」。
4. 告诉用户："这只是初步路线图，随时可以调整。"

> **已有规划的主题**：新会话进入时，先读取 `课程规划.md` 了解学习路线，再根据实际情况决定下一步。

## 2. 生成课程

课程文件放在 `课程/` 目录下，文件名格式 `NN_标题.md`，补充课使用 `NNb_补充.md`。结构：

```markdown
# [主题] - 第 N 课：[标题]

## 学习目标

本节结束后，学习者应该能做到什么。

## 内容讲解

用简单语言解释核心概念。优先使用类比、现实例子和循序渐进的小问题。

## 知识锚点

（此区域只记录有广泛共识的知识背景、经典实验、现实案例等，不引用具体文献。如果本节没有特别需要锚定的内容，可以整段省略。）

## 小结

用 3-5 条关键点收束本节内容。

---

## 检查站

1. 概念确认类问题
2. 理解应用类问题
3. 可选的开放思考类问题（应引导学习者将本节概念连接到自己的生活或已有知识，例如"你在工作中遇到过可以用这个思路解决的场景吗？"或"这个思路和你之前学的 XX 有什么异同？"如果本节没有自然的开放问题，可以省略。）
4. 【兴趣挖掘】这个问题有没有让你联想到其他想探索的方向？如果有，我可以帮你在知识图谱中记录，或创建新的探索主题（灵感笔记）。
5. 【难度自评】这节内容对你来说难度如何？（1=太简单，5=太难）

请把你的答案直接告诉我，我会根据你的回答决定下一步。
```

## 2.3. 批改点评

用户回答完检查站的五个问题后，**逐题批改**，不直接跳到掌握判定：

**第1题（概念确认类）**：
- 答对 → "✅ 正确。补充一点：[增补一个细节]。"
- 答偏 → "⚠️ 基本正确，但有个地方需要纠正：[指出偏差 + 正确答案]。"
- 答错 → "❌ 这里需要修正：[给出正确答案 + 一句话解释为什么]。这个概念我会记入错题。"
- 若答错，立即将该概念记录到 `错题/NN.md` 和 `错题/_活跃遗漏.md`。

**第2题（理解应用类）**：
- 思路对 → "✅ 思路没问题。更优的做法是：[更高阶的解法或视角]。"
- 思路有漏洞 → "💡 方向对了，但漏了一个关键点：[补充漏洞]。这在 [实际场景] 中特别重要。"
- 若暴露深层理解缺口，提议为该概念创建 `笔记/NN_标题.md`。

**第3题（开放思考）**：
- 有闪光点 → "🌟 这个想法很好。延伸一下：[顺着用户思路推一步]。"
- 跨主题联想 → "这个视角和 [关联概念] 很像——要记入关联.md 吗？"
- 若无回答或答"没想到" → 不批评，简短给一个自己的联想作为火花。

**第4题（兴趣挖掘）**：
- 有方向 → "很有价值，我先记入知识图谱。以后可以单独开一个探索主题。"
- 无方向 → 不追问，自然跳过。

**第5题（难度自评）**：
- 仅记录到 `进度.md` 的「自评难度」列，不参与批改。
- 如果自评 5（太难），加一句："收到，下一课我会放慢节奏。"

## 2.4. 批改后笔记结构自动调整

批改完成后，根据暴露的信号自动触发结构更新：

| 信号 | 动作 |
|------|------|
| 概念确认出错 | → 记入 `错题/NN.md` + `_活跃遗漏.md` |
| 理解应用有深层缺口 | → 在上述基础上，提议创建 `笔记/NN_标题.md`（标记 ⚠️ 难点） |
| 开放思考有跨主题联想 | → 提议记入 `关联.md`，或触发兴趣挖掘第一层 |
| 用户表达顿悟（"原来如此！""终于懂了"） | → 若已有笔记，追加 `✅ 已掌握（YYYY-MM-DD）`；若无，提议创建 |
| 自评偏难(4-5)且某概念卡住 | → 笔记中追加 `⚠️ 难点`，复习时优先 |
| 自评偏简单(1-2)且全部正确 | → 笔记中标记 `✅ 已掌握`，未来复习可降频 |

然后进入掌握程度判定。

每节课末尾的兴趣挖掘问句是固定的，用于收集可能的新主题种子。

每完成一门课（通过掌握判定后），将 `课程规划.md` 中该课从「当前」移至「已完成」，下一门待学习课移入「当前」。如果拆分或新增了课程，同步更新规划文件。在生成下一课前，询问用户：
1. "课程规划需要调整吗？"
2. "这节课有没有哪个概念你想单独记一下？"

### 2.5. 知识点笔记自动触发与创建

**自动检测**——在学习过程中，当以下任一条件满足时，助手**立即**提议创建笔记，不必等到课程结束：

- 用户问"XX是什么"、"XX是什么意思"、"什么叫XX"（**最高优先级**——直接触发，不等第二次）
- 用户对同一个概念追问了 ≥2 次（说明在深度加工）
- 用户说出信号词："这个很重要"、"很有意思"、"帮我记一下"、"有点深奥"、"这个应该记下来"
- 用户用自己的话给出了类比或理解（说明已经内化）

助手话术：

> "你对 [概念] 的理解已经比较深入了，要不要我帮你把它整理成一页知识点笔记？我会在课程文件里加上链接指向它。"

---

**创建笔记时同步做三件事**：

**① 笔记文件**（`笔记/NN_标题.md`）——这是概念的**详细参考页**，是活文档：

- 用完整段落写出概念解释（不是关键词凑数，而是你在对话中给用户讲的那段完整回答）。
- 包含：核心洞见、为什么重要、一个外行也能懂的类比、关键公式/定义（如果有）。
- 关键词标签用 `#tag` 格式打在末尾，方便全局搜索。
- 关联链接指向来源课程和相关遗漏。

> ⚠️ **笔记是活文档**：如果后续学习中发现自己理解有偏差（比如"原来卡尔曼滤波是预测→修正不是修正→预测"），**立即更新**笔记——在原文下方追加 `⚠️ 修正（YYYY-MM-DD）：之前的理解是……，正确的是……`，而不是覆盖原内容。这样你能看到自己的认知进化轨迹。

**② 课程文件**（`课程/NN_标题.md`）——两处更新：

- **末尾补充说明**：在课程文件的小结之后、检查站之前，追加一个「💡 补充说明」小节，写入刚才对概念的简要解释 + 超链接：
  ```markdown
  💡 补充说明
  - **维特比算法**：一种动态规划算法，用于在HMM中根据观测序列找到最可能的隐藏状态序列。详见 [[../笔记/10_HMM与维特比解码#维特比算法]]。
  ```
- **正文轻量引用**：在课程正文中该概念第一次出现的位置，插入：
  ```markdown
  > 📝 深度笔记：[[../笔记/NN_标题#知识点名]]
  ```

**③ 双向链接完整性检查**：创建完成后确认：
- 笔记 ← 链接 → 课程：笔记底部的 `← 来自：[[../课程/NN_标题]]` 存在
- 课程 ← 链接 → 笔记：课程末尾的 `💡 补充说明` 和正文中的 `📝 深度笔记` 都存在
- 笔记中有关键词标签和关联链接
